# job4j_sql

## Описание проекта

Учебный репозиторий для изучения SQL и проектирования баз данных. В рамках проекта создаётся схема интернет-магазина из четырёх таблиц: `users`, `products`, `orders`, `order_items`. Реализованы связи между таблицами, ограничения целостности, автоматическая генерация ID и хранение исторической цены товара в заказе.

---

## Схема БД

| Таблица | Описание | Основные поля |
|---------|----------|---------------|
| `users` | Пользователи | `id`, `name`, `email` (уникальный), `created_at` |
| `products` | Товары | `id`, `name`, `price` (>0), `is_active`, `created_at` |
| `orders` | Заказы | `id`, `user_id` (FK), `status`, `created_at` |
| `order_items` | Позиции заказа | `id`, `order_id` (FK), `product_id` (FK), `quantity` (>0), `unit_price` (>0) |

---

## Связи

- `users` → `orders`: один пользователь → много заказов (1:N)
- `orders` → `order_items`: один заказ → много позиций (1:N)
- `products` → `order_items`: один товар → много позиций (1:N)

---

## Ключевые решения

- **`unit_price`** хранит цену товара **на момент заказа**, чтобы изменения цены в каталоге не влияли на уже оформленные заказы
- **`GENERATED ALWAYS AS IDENTITY`** — автоматическая генерация первичных ключей (современный стандарт)
- **`CHECK`-ограничения** — гарантируют положительные значения цены и количества
- **Внешние ключи** — обеспечивают ссылочную целостность данных
